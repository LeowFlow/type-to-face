Process

The project was developed through research, wiremapping, and iterative prototyping. The process moved from typographic and computational references into a working browser-based system.

Research to Implementation

Generative Gestaltung was a key reference for understanding how computational rules can produce visual form. It provided a way to think about parameters, variation, and output as parts of one system.

Existing code was used as learning material rather than copied as a finished solution. The focus was on understanding how sampling, mapping, and rendering connect, then rebuilding those ideas around the requirements of Face-To-Type.

System Architecture

  1. Input The browser requests webcam access and treats the live stream as the image source.
  2. Image Sampling The video frame is reduced to a grid so each cell can be measured and translated.
  3. Typographic Mapping Brightness values are mapped to glyphs and typographic behaviour.
  4. Rendering Engine The mapped glyphs are drawn to canvas as a live typographic reconstruction.
  5. Configuration System Parameters such as cell size, density, text scale, and tracking remain adjustable.

Prototype Development

The first prototype reduced the system to a minimal viable form: webcam input to ASCII output.

This proof of concept tested whether a live face could be sampled, mapped, and rendered typographically before broader controls were added.

Iteration and Refinement

Refinement focused on adjustable parameters including cell size, density, text scale, and spacing.

Each adjustment involved a trade-off. Smaller cells increased facial detail but reduced typographic legibility, while larger cells made the letters clearer and pushed the image further into abstraction.

The process negotiated abstraction and clarity. The aim was not photographic accuracy, but a reconstruction that remained recognisably facial while foregrounding the type.
